# Гайд по форматированию текста TTG.CLUB (разметка `{@...}`)

Этот документ описывает **единый стандарт форматирования** описаний на сайте
(заклинания, существа, классы, черты, предыстории, предметы, глоссарий и т.д.).
Формат — собственная разметка вида `{@...}`. Обычный Markdown (`**жир**`, `#`,
`- пункт`, `| таблицы |`) **не используется** — всё оформление идёт только через
маркеры `{@...}`.

Документ пригоден для двух целей:

1. **Для авторов** — как правильно оформлять текст (разделы 1–9, примеры в 10).
2. **Для обучения нейросети** — точная грамматика, справочник значений и готовый
   системный промпт (разделы 8, 10, 11). Нейросеть должна выдавать **строку**
   `{@...}` (авторский формат) — редактор и бэкенд принимают её напрямую.

---

## 0. Шпаргалка (все теги на одной странице)

### Инлайн-форматирование (внутри абзаца)

| Что нужно | Пиши так | Результат |
|---|---|---|
| Жирный | `{@b текст}` | **текст** |
| Курсив | `{@i текст}` | *текст* |
| Подчёркнутый | `{@u текст}` | <u>текст</u> |
| Зачёркнутый | `{@s текст}` | ~~текст~~ |
| Верхний индекс | `{@sup текст}` | текст<sup>верх</sup> |
| Нижний индекс | `{@sub текст}` | текст<sub>низ</sub> |
| Выделение (маркер) | `{@mark текст}` | подсвеченный фон |
| Перенос строки | `{@br}` | мягкий перенос внутри абзаца |

### Инлайн-вставки

| Что нужно | Пиши так |
|---|---|
| Ссылка (внешняя/внутренняя) | `{@link текст \| url:https://...}` |
| Клавиша | `{@kbd Ctrl}` |
| Бейдж | `{@badge текст \| color:primary}` |
| Бросок кубика | `{@dice 1к10}` |

### Блочные элементы (отдельным блоком, отделяются пустой строкой)

| Что нужно | Пиши так |
|---|---|
| Заголовок (level 1–4) | `{@h Заголовок \| level:1}` |
| Маркированный список | `{@list {@li пункт 1}{@li пункт 2}}` |
| Нумерованный список | `{@list {@li пункт 1}{@li пункт 2} \| type:ordered}` |
| Цитата | `{@quote Текст цитаты}` |
| Таблица | `{@table {@tr {@th Загол.}}{@tr {@td Ячейка}}}` |
| Разделитель (линия) | `{@separator}` |

### Ссылки на разделы сайта (открывают карточку сущности) — **обязателен `url:`**

| Раздел | Пиши так |
|---|---|
| Заклинание | `{@spell Огненный шар \| url:fireball-phb}` |
| Существо | `{@creature Гоблин \| url:goblin-mm}` |
| Класс | `{@class Волшебник \| url:wizard-phb}` |
| Черта | `{@feat Меткий стрелок \| url:sharpshooter-phb}` |
| Предыстория | `{@background Солдат \| url:soldier-phb}` |
| Магический предмет | `{@magicItem Сумка хранения \| url:bag-of-holding-dmg}` |
| Предмет (снаряжение) | `{@item Верёвка \| url:rope-phb}` |
| Глоссарий (термин) | `{@glossary тусклый свет \| url:dim-light-phb}` |

> В таблицах выше `\|` — это экранированная вертикальная черта только для
> отображения в Markdown. В реальном тексте пиши обычную `|`.

## 1. Что это и где применяется

Разметка `{@...}` используется во всех полях **«Описание»** в редакторах контента
(заклинания, бестиарий, классы, виды, черты, предыстории, магические предметы,
снаряжение, глоссарий, статьи и т.п.).

Редактор работает в двух режимах, дающих **один и тот же результат**:

- **Визуальный (WYSIWYG)** — выделяешь текст и жмёшь кнопку на панели
  (жирный, список, таблица, вставка ссылки на заклинание и т.д.).
- **Код** — редактируешь исходную строку `{@...}` руками (кнопка «Показать код»).

Этот гайд описывает именно **код-формат** — то, что видно в режиме кода и что
должна порождать нейросеть.

## 2. Как устроен маркер (синтаксис)

### 2.1. Общая форма

```
{@имя тело | атрибут:значение | атрибут:значение}
```

- Всё начинается с `{@`, заканчивается `}`.
- `имя` — тип маркера (`b`, `i`, `spell`, `list`, …). Отделяется от тела
  **первым пробелом**.
- `тело` — содержимое (текст или вложенные маркеры).
- `атрибуты` — необязательный «хвост» после `|`. Каждый атрибут — пара
  `ключ:значение`. Несколько атрибутов разделяются `|`. Разделитель ключа и
  значения — двоеточие `:` (канонично; парсер принимает и `ключ=значение`, но
  пиши `:`).

Примеры:

```
{@b важно}
{@h Способности | level:2}
{@spell Огненный шар | url:fireball-phb}
{@badge Ядовитый | color:success | variant:soft}
```

### 2.2. Абзацы

Абзацы разделяются **пустой строкой** (двойным переносом). Каждый абзац — это
отдельная строка с инлайн-маркерами. Блочные элементы (список, таблица, цитата,
заголовок) тоже отделяются пустой строкой от соседних абзацев.

```
Первый абзац с {@i курсивом}.

Второй абзац.

{@h Заголовок раздела | level:1}

Абзац после заголовка.
```

Чтобы сделать перенос строки **внутри** одного абзаца (без нового абзаца) —
используй `{@br}`.

### 2.3. Вложенность

Маркеры можно вкладывать друг в друга (до **7 уровней** вложенности):

```
{@b Жирный и {@i ещё курсив} текст}
{@quote Цитата со {@spell ссылкой | url:bless-phb} внутри}
```

### 2.4. Канонические имена и синонимы (алиасы)

У большинства тегов есть короткое и длинное имя — они равнозначны:

| Канон | Синонимы |
|---|---|
| `bold` | `b` |
| `italic` | `i` |
| `underline` | `u` |
| `strikethrough` | `s` |
| `superscript` | `sup` |
| `subscript` | `sub` |
| `highlight` | `mark` |
| `break` | `br` |
| `link` | `a`, `link` |
| `roll` | `dice` |
| `heading` | `h` |
| `quote` | `blockquote`, `q` |
| `li` | `listItem` |
| `magicItem` | `magic-item` |
| `creature` | `bestiary` |

**Рекомендация:** пиши короткие формы (`{@b}`, `{@i}`, `{@h}`, `{@dice}`) — их
вставляют кнопки редактора, и именно так выглядит большинство контента.

### 2.5. Запрещённые символы (важно!)

В грамматике `{@...}` **нет экранирования**. Символы `{`, `}` и `|` внутри тела
маркера имеют служебный смысл и **ломают разбор**. Поэтому:

- Не пиши литеральные `{` `}` внутри тела маркера.
- Символ `|` внутри тела воспринимается как начало атрибутов. Если он нужен как
  обычный текст — по возможности замени на `/` или вынеси из маркера.

Редактор при вставке автоматически чистит подписи: `|` → `/`, а `{` и `}`
удаляет. Нейросеть должна делать то же самое: **не помещать `{`, `}`, `|` внутрь
тела маркера как обычные символы.**

## 3. Инлайн-форматирование текста

Применяется к части текста внутри абзаца.

| Тег | Назначение | Пример |
|---|---|---|
| `{@b …}` | Жирный | `Урон {@b удваивается}` |
| `{@i …}` | Курсив | `{@i дальнобойная атака заклинанием}` |
| `{@u …}` | Подчёркнутый | `{@u обязательное условие}` |
| `{@s …}` | Зачёркнутый | `{@s старое значение}` |
| `{@sup …}` | Верхний индекс | `1-й уровень{@sup *}` |
| `{@sub …}` | Нижний индекс | `H{@sub 2}O` |
| `{@mark …}` | Выделение фоном | `{@mark ключевое слово}` |
| `{@br}` | Перенос строки | `Строка 1{@br}Строка 2` |

Форматирование вкладывается: `{@b {@i жирный курсив}}`.

## 4. Инлайновые вставки

### 4.1. Ссылка `{@link}`

```
{@link текст ссылки | url:https://example.com}
{@link раздел сайта | url:/spells}
```

- `url` — адрес. Может быть внешним (`https://…`) или внутренним путём (`/spells`).
- Внешние ссылки и `| target:_blank` открываются в новой вкладке.
- Если `url` пустой — текст покажется как обычный (ссылка не сломает страницу).

> Для ссылок на сущности сайта (заклинания, существа и т.д.) используй
> **специальные теги раздела** из §6, а не `{@link}`.

### 4.2. Клавиша `{@kbd}`

Оформляет нажатие клавиши:

```
Нажмите {@kbd Ctrl} + {@kbd S}
```

### 4.3. Бейдж `{@badge}`

Небольшая цветная плашка:

```
{@badge Концентрация | color:warning}
{@badge Ритуал | color:info | variant:soft}
```

Атрибуты: `color`, `variant`, `size` (см. §7).

### 4.4. Бросок кубика `{@dice}`

Кликабельная нотация кубика — бросает кости при нажатии:

```
Цель получает {@dice 1к10} урона огнём.
Восстанавливает {@dice 2к6+3} хитов.
```

- Нотация — стандартная запись кубиков: `1к10`, `2к6`, `1к20+5`
  (русская `к` = «кубик»; латинская `d`, например `2d6`, тоже работает).
- Синоним — `{@roll …}` (так тег хранится в базе; при вводе пиши `{@dice …}`).
- Внутри **таблиц** бросок в шапке первого столбца подсвечивает строку с выпавшим
  значением (диапазоны вида `01-40`, `41-75` в первом столбце).

## 5. Блочные элементы

Каждый блочный элемент стоит **отдельно** и отделяется пустой строкой от абзацев.

### 5.1. Заголовки `{@h}`

```
{@h Главный заголовок | level:1}
{@h Подзаголовок | level:2}
{@h Мелкий заголовок | level:3}
{@h Минимальный заголовок | level:4}
```

- `level` — от `1` до `4`:
  - `1` — крупный заголовок раздела (с нижней линией);
  - `2` — подзаголовок;
  - `3` — мелкий раздел;
  - `4` — минимальный (капсом).
- Если `level` не указан — по умолчанию `1`.
- Кнопки редактора вставляют H1/H2/H3; `level:4` доступен вручную.

### 5.2. Списки `{@list}` / `{@li}`

Список — это `{@list}`, внутри которого пункты `{@li …}`:

```
{@list {@li Первый пункт}{@li Второй пункт}{@li Третий пункт}}
```

Нумерованный список — добавь `| type:ordered`:

```
{@list {@li Шаг один}{@li Шаг два} | type:ordered}
```

Без `type` (или `type:unordered`) список маркированный (с точками).

**Вложенные списки** — вставь `{@list}` прямо внутрь `{@li}` (или рядом с ним):

```
{@list {@li Пункт верхнего уровня}{@list {@li Подпункт 1}{@li Подпункт 2}}{@li Ещё пункт}}
```

Пункт может содержать любое инлайн-форматирование и ссылки:

```
{@list {@li Урон {@dice 1к6}}{@li См. {@spell Благословение | url:bless-phb}}}
```

### 5.3. Цитаты `{@quote}`

Выделенный блок-врезка:

```
{@quote Дракон медленно открыл глаз. «Ты опоздал», — прошипел он.}
```

Несколько абзацев внутри цитаты разделяй двойным `{@br}{@br}`:

```
{@quote Первый абзац цитаты.{@br}{@br}Второй абзац цитаты.}
```

Необязательные атрибуты оформления (задаются в режиме кода):

- `color` — цвет (см. §7), по умолчанию `neutral`;
- `variant` — вид (см. §7), по умолчанию `soft`.

```
{@quote Важное предупреждение! | color:warning | variant:soft}
```

### 5.4. Таблицы `{@table}`

Таблица состоит из строк `{@tr}`. Ячейки:

- `{@th …}` — ячейка **шапки** (заголовок столбца);
- `{@td …}` — ячейка **тела**.

Первая строка с `{@th}` — это шапка; остальные строки из `{@td}` — тело.

```
{@table {@tr {@th Литера}{@th Значение}}{@tr {@td 1к4}{@td Слабый}}{@tr {@td 1к6}{@td Сильный}}}
```

**Атрибуты ячеек:**

- `{@th … | style:col-3}` — CSS-классы столбца (например ширина `col-3`);
  применяются ко всему столбцу.
- `{@th … | align:left}` / `{@td … | align:right}` — выравнивание текста:
  `left`, `center`, `right`. Шапка по умолчанию центрирована.

**Подпись таблицы** — первым элементом `{@caption …}`:

```
{@table {@caption Таблица случайных сокровищ}{@tr {@th d100}{@th Награда}}{@tr {@td 01-50}{@td 10 зм}}{@tr {@td 51-00}{@td Самоцвет}}}
```

**Бросок по таблице:** если в шапке первого столбца стоит `{@dice}`, а в первом
столбце — числа/диапазоны (`01-50`, `51-00`), то бросок подсветит выпавшую строку.
`00` трактуется как `100`.

> Пустую ячейку оставляй как `{@td }` (редактор сам подставит невидимый
> символ-заглушку) — не оставляй ячейку совсем без содержимого.

### 5.5. Разделитель `{@separator}`

Горизонтальная линия-разделитель:

```
{@separator}
{@separator | color:info}
```

- Атрибут `color` задаёт цвет линии (см. §7).
- В визуальном тулбаре кнопки нет — вставляется в **режиме кода**.

## 6. Ссылки на разделы сайта

Эти теги превращают текст в ссылку, открывающую карточку сущности (дровер) и
ведущую на её страницу. **Все они требуют атрибут `url` — это slug (адрес)
сущности.**

| Тег | Раздел | Куда ведёт |
|---|---|---|
| `{@spell Название \| url:slug}` | Заклинания | `/spells/slug` |
| `{@creature Название \| url:slug}` | Бестиарий | `/bestiary/slug` |
| `{@class Название \| url:slug}` | Классы | `/classes/slug` |
| `{@feat Название \| url:slug}` | Черты | `/feats/slug` |
| `{@background Название \| url:slug}` | Предыстории | `/backgrounds/slug` |
| `{@magicItem Название \| url:slug}` | Магические предметы | `/magic-items/slug` |
| `{@item Название \| url:slug}` | Снаряжение | `/items/slug` |
| `{@glossary термин \| url:slug}` | Глоссарий | `/glossary/slug` |

Пример из реального контента:

```
Существо в зоне {@glossary тусклого света | url:dim-light-phb} получает помеху.
```

**Где взять `url` (slug):** это поле `url` сущности. В визуальном редакторе кнопка
«Ссылка на раздел» открывает поиск — выбери сущность, и `url` подставится сам.
Slug обычно выглядит как `fireball-phb`, `goblin-mm`, `dim-light-phb`
(название + суффикс источника).

> ⚠️ **Критическое правило.** Ссылка на раздел **без** `url:` (пустой или
> отсутствующий) **ломает рендер всего описания**. Если slug неизвестен — **не
> ставь** тег раздела: используй курсив `{@i название}` или обычный текст.
> Это особенно важно для нейросети — см. §9 и §11.

## 7. Справочник значений атрибутов

| Атрибут | Где применяется | Допустимые значения | По умолчанию |
|---|---|---|---|
| `level` | `{@h}` | `1`, `2`, `3`, `4` | `1` |
| `type` | `{@list}` | `ordered`, `unordered` | `unordered` |
| `color` | `{@quote}`, `{@badge}`, `{@kbd}`, `{@separator}` | `primary`, `secondary`, `success`, `info`, `warning`, `error`, `neutral` | `neutral` |
| `variant` | `{@quote}`, `{@badge}`, `{@kbd}` | `solid`, `outline`, `soft`, `subtle` | `soft` |
| `size` | `{@badge}`, `{@kbd}` | `sm`, `md`, `lg` | `md` |
| `align` | `{@th}`, `{@td}` | `left`, `center`, `right` | шапка — `center`, тело — `left` |
| `style` | `{@th}` | CSS-классы столбца (напр. `col-3`) | — |
| `url` | `{@link}` и все теги раздела (§6) | адрес/slug | — (обязателен для §6) |
| `target` | `{@link}` | `_blank` (новая вкладка) | — |

> Недопустимое значение цвета/варианта/размера не ломает страницу — подставляется
> значение по умолчанию. Но лучше сразу писать корректно.

## 8. Как это хранится (для бэкенда и нейросети)

Важно различать **две формы**:

1. **Авторская строка** (то, что ты пишешь и что должна выдавать нейросеть):
   единая строка с `{@...}` и абзацами через пустую строку. Пример:

   ```
   Вы метаете частицу огня. Совершите {@i дальнобойную атаку заклинанием}. Цель получает {@dice 1к10} урона огнём.

   {@list {@li Первый эффект}{@li Второй эффект}}
   ```

2. **Форма хранения (AST)** — редактор автоматически превращает строку в
   JSON-массив: абзацы остаются строками, а блоки (`list`/`table`/`quote`/
   `heading`/`separator`) становятся объектами-узлами. Пример того же описания в
   базе:

   ```json
   [
     "Вы метаете частицу огня. Совершите {@i дальнобойную атаку заклинанием}. Цель получает {@roll 1к10} урона огнём.",
     {
       "type": "list",
       "attrs": { "type": "unordered" },
       "content": [ "Первый эффект", "Второй эффект" ]
     }
   ]
   ```

**Для нейросети целевой формат — авторская строка (форма 1).** Не нужно
генерировать JSON-AST: редактор и бэкенд сами разберут строку `{@...}`.
Внутри инлайн-элементов допускается `{@roll …}` — это то же самое, что `{@dice …}`.

## 9. Правила и типичные ошибки (чеклист)

✅ **Делай так:**

- Абзацы — через пустую строку; перенос внутри абзаца — `{@br}`.
- Блоки (список/таблица/цитата/заголовок) — отдельными блоками, с пустой строкой
  вокруг.
- Ссылки на сущности — только с реальным `url:` slug.
- Короткие имена тегов (`{@b}`, `{@i}`, `{@h}`, `{@dice}`).
- Каждый `{@...}` закрыт `}`; число `{` равно числу `}`.

❌ **Не делай так:**

- ❌ Markdown: `**жир**`, `# заголовок`, `- пункт`, `| таблица |`,
  `[текст](ссылка)` — **не поддерживается**, пиши через `{@...}`.
- ❌ Ссылка на раздел без `url:` → **ломает всё описание**.
- ❌ Литеральные `{`, `}`, `|` внутри тела маркера.
- ❌ Незакрытый или несбалансированный маркер (`{@b текст` без `}`).
- ❌ Пункты списка без обёртки `{@li …}`.
- ❌ Ячейки таблицы вне `{@tr}`; строки без `{@th}`/`{@td}`.

**Самая частая и опасная ошибка** — тег раздела (`{@spell}`, `{@creature}` …) без
`url:`. Если сомневаешься в slug — не ставь ссылку.

## 10. Примеры «обычный текст → размеченный» (для обучения)

Пары «вход → выход». Вход — сырой текст, выход — корректная разметка `{@...}`.

**Пример 1. Инлайн-форматирование и кубик**

Вход:
```
Вы метаете частицу огня в существо в пределах дистанции. Совершите дальнобойную атаку заклинанием против цели. При попадании цель получает 1к10 урона огнём.
```
Выход:
```
Вы метаете частицу огня в существо в пределах дистанции. Совершите {@i дальнобойную атаку заклинанием} против цели. При попадании цель получает {@dice 1к10} урона огнём.
```

**Пример 2. Заголовок + маркированный список**

Вход:
```
Особенности:
- Тёмное зрение 18 метров
- Сопротивление урону огнём
- Владение языком Общий и Драконий
```
Выход:
```
{@h Особенности | level:2}

{@list {@li Тёмное зрение 18 метров}{@li Сопротивление урону огнём}{@li Владение языком Общий и Драконий}}
```

**Пример 3. Нумерованный список с вложенностью**

Вход:
```
Порядок хода:
1. Объявите действие
2. Совершите бросок атаки
   - При попадании — бросок урона
   - При промахе — ход переходит дальше
3. Завершите ход
```
Выход:
```
{@list {@li Объявите действие}{@li Совершите бросок атаки}{@list {@li При попадании — бросок урона}{@li При промахе — ход переходит дальше}}{@li Завершите ход} | type:ordered}
```

**Пример 4. Таблица с броском**

Вход:
```
Таблица случайных эффектов (бросьте d100):
01-50 — Ничего не происходит
51-90 — Лёгкое свечение
91-00 — Взрыв на 3к6 урона
```
Выход:
```
{@table {@caption Случайные эффекты}{@tr {@th {@dice 1к100}}{@th Эффект}}{@tr {@td 01-50}{@td Ничего не происходит}}{@tr {@td 51-90}{@td Лёгкое свечение}}{@tr {@td 91-00}{@td Взрыв на {@dice 3к6} урона}}}
```

**Пример 5. Цитата**

Вход:
```
Надпись на камне гласит: «Только достойный пройдёт дальше».
```
Выход:
```
{@quote Только достойный пройдёт дальше.}
```

**Пример 6. Ссылка на заклинание (slug известен)**

Вход:
```
Существо действует так, будто на него наложено заклинание Благословение (slug: bless-phb).
```
Выход:
```
Существо действует так, будто на него наложено заклинание {@spell Благословение | url:bless-phb}.
```

**Пример 7. Ссылка на сущность, когда slug неизвестен**

Вход:
```
Цель получает состояние «Отравление».
```
Выход (slug не дан — ссылку не выдумываем, оформляем курсивом):
```
Цель получает состояние {@i Отравление}.
```

**Пример 8. Бейдж и клавиша**

Вход:
```
Заклинание требует концентрации. Для быстрого броска нажмите пробел.
```
Выход:
```
Заклинание требует {@badge Концентрация | color:warning}. Для быстрого броска нажмите {@kbd Пробел}.
```

## 11. Готовый системный промпт для нейросети

Скопируй этот блок как системную инструкцию модели, которая переводит обычный
текст в разметку TTG.

```
Ты — форматировщик текста для сайта TTG.CLUB (D&D-контент на русском). Тебе дают
обычный текст, ты возвращаешь тот же текст, размеченный собственным форматом сайта
{@...}. НЕ используй Markdown. Верни ТОЛЬКО размеченный текст, без пояснений.

ПРАВИЛА РАЗМЕТКИ:
- Формат маркера: {@имя тело | атрибут:значение}. Имя от тела отделяет первый
  пробел; атрибуты — после |, в виде ключ:значение.
- Абзацы разделяй пустой строкой. Перенос внутри абзаца — {@br}.
- Внутри тела маркера НЕ ставь символы { } | (они ломают разбор); при
  необходимости | заменяй на /.

ИНЛАЙН-ФОРМАТИРОВАНИЕ:
- Жирный {@b …}; курсив {@i …}; подчёркнутый {@u …}; зачёркнутый {@s …};
  верхний индекс {@sup …}; нижний индекс {@sub …}; выделение {@mark …}.
- Игровые термины и названия эффектов/состояний оформляй курсивом {@i …}, если
  для них не дан slug ссылки.

КУБИКИ:
- Любую нотацию кубика (1к10, 2к6, 1к20+5, 3к6, d100) оборачивай в {@dice …}.

БЛОКИ (каждый — отдельным блоком, с пустой строкой вокруг):
- Заголовок: {@h Текст | level:N}, где N = 1..4 (1 — крупный, 4 — мелкий).
- Маркированный список: {@list {@li пункт}{@li пункт}}.
- Нумерованный список: {@list {@li пункт}{@li пункт} | type:ordered}.
- Вложенный список: вставь {@list ...} внутрь {@list ...}.
- Цитата: {@quote текст}. Несколько абзацев — через {@br}{@br}.
- Таблица: {@table {@tr {@th Загол.}{@th Загол.}}{@tr {@td Ячейка}{@td Ячейка}}}.
  Первая строка {@tr} с {@th} — шапка; далее {@tr} с {@td} — тело. Подпись —
  {@caption …} первым элементом. Выравнивание ячейки — | align:left|center|right.
- Разделитель: {@separator}.

ССЫЛКИ НА РАЗДЕЛЫ САЙТА (открывают карточку сущности) — ТОЛЬКО если дан slug (url):
- {@spell Название | url:slug}      — заклинание
- {@creature Название | url:slug}   — существо
- {@class Название | url:slug}       — класс
- {@feat Название | url:slug}        — черта
- {@background Название | url:slug}  — предыстория
- {@magicItem Название | url:slug}   — магический предмет
- {@item Название | url:slug}         — снаряжение
- {@glossary термин | url:slug}       — термин глоссария
- Внешняя ссылка: {@link текст | url:https://...}.

КРИТИЧЕСКОЕ ПРАВИЛО: НИКОГДА не создавай тег раздела без реального url — это ломает
всю страницу. Если slug неизвестен — используй {@i название} или обычный текст.

ЗАПРЕЩЕНО: **жир**, # заголовки, - списки, | таблицы |, [текст](url) и любой
другой Markdown. Только {@...}.
```
